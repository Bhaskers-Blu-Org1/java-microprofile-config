Welcome to the MicroProfile Config demonstrator application

Please see below for some information on each of the technologies used in this application.

MicroProfile Config

Inside the project there is an application.rest package containing the ConfigRestEndpoint class. This adds a REST endpoint which you can access at rest/config. It will return a JSON representation of all of the available configuration properties. An individual configuration property can be accessed by adding the property name at the end of the URL. For example rest/config/java.home.

The project also includes an application.configsource package containing the TimeConfigSource class. This class is a ConfigSource implementation which returns the current time under the property name time.now.

The project also includes an application.converter package containing the PatternConverter and ResourceBundleConverter classes. These classes is are Converter implementations which convert property values from type String to the type supported by the converter.

Finally, the application.rest package contains the LibertyRestEndpoint class. This adds a REST endpoint which you can access at rest. This class demonstrates using dependency injection to receive configuration properties as well as conversion through the Converter implementations as well as using javax.inject.Provider to get dynamic property values like time.now.

There is also a test classes named it.config.ConfigRestEndpointTest and it.rest.LibertyRestEndpointTest that will test the REST endpoints to ensure they is working.

For information on the Config feature of MicroProfile, see MicroProfile Config.
